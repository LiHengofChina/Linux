
/**
磁碟阵列(RAID)是利用硬件技术将数个硬碟
整合成为一个大硬碟的方法，
操作系统，只会看到最后被整合起来的大硬碟。
*/

 
1). 在Linux系统中，每个装置(指设备:鼠标、硬盘等等)
	都被当成一个档案(文件)来对待
	
	//所以设备对应的文件名字都在 /dev 目录下
	

2). "一块磁碟"可以被分割成"多个分割槽(partition)"

	/**
	一块磁碟是可以被分割成多个分割槽的(partition)
	
	以Windows为例
	一个磁碟被分为：C:, D:, E: 盘
	
	那个C, D, E就是分割槽(partition)
 
	*/
	
3). linux中磁碟的装置档名

	/dev/sd[ap]
	/dev/vd[ap] //如果是虚拟机，一般是这种

	// 工作中经常碰到的 /dev/vda  就是虚拟的

 
4). 磁碟盘中的两种单位
		磁区(Sector)  //也叫：扇区	512bytes与4Kbytes
		与磁轨(Track) //
				同一磁轨上的磁区形成一个磁柱


5). 整颗磁碟的"第一个磁区" 最重要
		它记录了整颗磁碟的重要信息。
		


6). 磁盘分割
		1). 早期： MSDOS (MBR) 分割		// fdisk 命令
			MBR使用第一个512bytes区块来记录"分割信息"
			1). 主要开机记录区(Master Boot Record, MBR)//446 bytes
			2). 分割表(partition table)//64 bytes
					//  最多只能容纳四笔分割的记录
					//  以磁柱为分割单位
					//  它们被称为主要(Primary)或扩展(Extended)分割槽。 
					//  扩展分割槽上面再分割出逻辑分割区。
					//  扩展分配的目的是"使用额外的磁区来记录分割资讯",
					//  这样可以创建更多的分割槽
		2). 近期： GPT(GUID partition table)分割	// gdisk 或 parted 命令
				/**
				旧磁区512bytes、新磁区4K	//
				为了兼容新老磁盘
				产生了"逻辑区块位址(Logical Block Address, LBA)概念"
				*/
			1). GPT使用34个LBA来记录"分割信息"
			2). GPT拿最后33个LBA作为备份
			3).
				LBA0 (与MBR模式相似的)
					开机管理程序 446 bytes
					特殊标志的，表示为为GPT格式 64 bytes
				LBA1 
					//(GPT 表头纪录)
				LBA2-33
					//共32个LAB，32*4=128笔分割记录
			4). GPT 分割已经没有所谓的主、延伸、逻辑分割的概念
				每笔纪录都可以独立存在
				每笔纪录都可以视为是主分割
				每一个分割都可以拿来格式化使用
		//==================================其它
			1). fdisk  不认识GPT
				要使用 gdisk  或 parted  才能进行 GPT 分割
			2). grub2 以后才认识GPT
			3). 操作系统不一定都能读取GPT 的分割
			4). 也不是所有硬件都能使用GPT分割
